## Import Conventions

- Prefer domain-first imports for types and avoid hub imports in new code
- Examples:

```typescript
// Types
import { User, UserType } from 'src/types/users';
import { Role } from 'src/types/rbac';
import { Book, BookStatus } from 'src/types/books';
import { Loan, BorrowStatus } from 'src/types/circulation';
import { Notification, NotificationType } from 'src/types/notifications';

// Legacy compatibility (allowed but not preferred)
// import * as Entities from 'src/types/entities';
```

## Features Overview

### Reference Books System
- **Purpose**: Manage reference materials and digital resources
- **Key Features**:
  - Digital book management
  - Category organization
  - Search and filtering
  - Access control
- **Technology**: Cloudflare D1 database with React frontend

### Email Service
- **Purpose**: Send notifications and communications to users
- **Architecture**: Direct Mailjet integration
- **Types**: Account and transactional emails via Mailjet

### Action Logging System
- **Purpose**: Comprehensive audit trail and system monitoring
- **Key Features**:
  - Automatic action capture
  - Security and audit features
  - Advanced filtering and search
  - Role-based access control
- **Technology**: Cloudflare D1 database with comprehensive logging

### Permissions System
- **Purpose**: Role-based access control with performance optimization
- **Key Features**:
  - Zustand-powered state management
  - Selective subscriptions for optimal performance
  - 100% API compatibility with original hook
  - Comprehensive permission management UI
  - Audit logging for security
- **Technology**: React hooks with Zustand store, TypeScript support

### Dashboard System
- **Purpose**: Single, config-driven dashboard for all user roles
- **Key Features**:
  - Unified `/dashboard` route
  - Role-specific widget layouts
  - Widget registry for reuse
  - Superuser config editor
  - Permission-based widget filtering
- **Technology**: React, TypeScript, centralized config files

### Badge System
- **Purpose**: Gamification and achievement recognition system
- **Key Features**:
  - Automatic badge awarding based on user actions
  - Manual badge assignment by administrators
  - Role-based virtual badges
  - Badge expiration and revocation
  - Comprehensive security controls
  - Full audit logging
- **Technology**: Cloudflare D1 database with React frontend

### Digital Resources
- **Purpose**: Manage and read e-books, PDFs, and online materials
- **Key Features**:
  - In-app PDF reader
  - Read session tracking
  - Multiple download sources
  - Resource type management
  - Activity analytics
- **Technology**: Cloudflare R2 storage, React PDF viewer, D1

### User Notes
- **Purpose**: Private note-taking for users
- **Key Features**:
  - Rich text notes
  - Revision history with restore
  - Soft delete and trash bin
  - Dashboard widget
- **Technology**: Cloudflare D1 with React frontend

### Issue Reports
- **Purpose**: Community-driven bug reporting and feature requests
- **Key Features**:
  - Public issue list
  - Upvotes
  - Image attachments
  - GitHub sync
- **Technology**: Cloudflare D1, GitHub API, R2

## Best Practices

### Reference Books Management
- **Organize by categories** for easy navigation
- **Use consistent naming** for files and metadata
- **Implement search functionality** for quick access
- **Control access** based on user permissions

### Email Service
- **Use direct Mailjet** for account-related emails
- **Use queues** for transactional emails
- **Implement proper error handling** for reliability
- **Monitor delivery rates** for optimization

### Action Logging System
- **Log all meaningful actions** for comprehensive audit trail
- **Use appropriate action types** for clear categorization
- **Implement proper error handling** for logging reliability
- **Monitor log performance** for system optimization

### Dashboard System
- **Keep configs serializable** — no JSX in config files
- **Register widgets centrally** before using them in configs
- **Use permission checks** to hide widgets users cannot access
- **Test layout changes** in a non-production environment first
