## Overview

Reservations let you join a waitlist when a book is currently unavailable (all copies are out). When a copy becomes available, the next reservation in the queue becomes "Ready" and can be claimed to start a loan.

**Route**: `/dashboard/reservations`

## Reservation Statuses

| Status | Color | Description |
|--------|-------|-------------|
| Pending | orange | Waiting in queue for a copy to become available |
| Ready | green | A copy is available — claim it to start a loan |
| Claimed | blue | Reservation has been converted to a loan |
| Expired | red | The ready reservation was not claimed in time |
| Cancelled | default | Reservation was cancelled by the user or staff |

## For Students & Regular Users

### Viewing Your Reservations

1. Navigate to **Dashboard** → **Reservations** (`/dashboard/reservations`)
2. You'll see a table with your reservations, including book title, status, queue position, and actions
3. Use the **status filter** dropdown to filter by status (e.g., show only "Ready" or "Pending")
4. Use the **search bar** to find reservations by book title or author

### Reserving a Book

There are two ways to reserve a book:

**From the Catalog:**
1. Go to **Dashboard** → **Catalog** (`/dashboard/catalog`)
2. Find the book you want
3. Click the book card to open details
4. Click **Reserve** to place a reservation
5. If you already have an active loan for this book, you'll see a message instead

**From the Reservations Page:**
1. Go to **Dashboard** → **Reservations**
2. Click **Create Reservation**
3. In the modal, search and select a book from the dropdown
4. Click **Create Reservation** to submit

### Claiming a Ready Reservation

When a copy becomes available, your reservation status changes to "Ready":

1. Go to **Dashboard** → **Reservations**
2. Find the reservation with status "Ready"
3. Click the **Claim** button (checkmark icon)
4. The reservation is converted to a loan with status "Pending" (awaiting staff approval)

<Aside type="caution" title="Claim Window">
Ready reservations have an expiration date. If you don't claim before it expires, the reservation moves to "Expired" and the next person in the queue gets the copy.
</Aside>

### Cancelling a Reservation

1. Go to **Dashboard** → **Reservations**
2. Find a reservation with status "Pending" or "Ready"
3. Click the **Cancel** button (X icon)
4. The reservation status changes to "Cancelled"

### Viewing Reservation Details

1. Click the **View** button (eye icon) on any reservation
2. The detail page shows:
   - Reservation ID and status
   - Book title and author
   - Your queue position and how many people are ahead
   - Expiration date (if Ready)
   - Notification date (when you were notified)
   - Created and updated timestamps

## For Librarians, Admins & Superusers

### Viewing All Reservations

1. Navigate to **Dashboard** → **Reservations**
2. The table shows all reservations across all users, with user columns visible
3. Use filters and search as described above

### Managing Reservations

Staff can:
- **View** any reservation's details
- **Claim** a ready reservation on behalf of a user
- **Cancel** any pending or ready reservation

### Creating Reservations for Users

1. Click **Create Reservation**
2. Select a book from the dropdown
3. Click **Create Reservation** to submit

## Queue System

When multiple users reserve the same book:
- Each reservation gets a **position** in the queue
- **Queue Ahead** shows how many people are ahead of you
- When a copy is returned, the first pending reservation becomes "Ready"
- The user is notified (via in-app notifications) that their reservation is ready
- If they don't claim before the expiration, it moves to "Expired" and the next person gets it

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/reservations` | List reservations (user-scoped or all) |
| GET | `/api/v1/reservations/:id` | Get reservation details |
| POST | `/api/v1/reservations` | Create a reservation |
| POST | `/api/v1/reservations/:id/claim` | Claim a ready reservation |
| POST | `/api/v1/reservations/:id/cancel` | Cancel a reservation |
