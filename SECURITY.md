# Security Policy

For the complete security policy and vulnerability reporting guidelines, please visit the [SJRS LMS Documentation](https://docs.sjrslms.in/policies/security-policy).

## Reporting a Vulnerability

If you believe you found a security vulnerability in SJRS LMS or its documentation, please report it to the project maintainers as soon as possible.

Preferred reporting methods:

- **GitHub Security Advisories** (if enabled): [https://github.com/PC-Mender/sjrslms/security/advisories](https://github.com/PC-Mender/sjrslms/security/advisories)
- **GitHub Issues**: [https://github.com/PC-Mender/sjrslms/issues](https://github.com/PC-Mender/sjrslms/issues)
  - Include `security` in the title and request that the issue be handled confidentially if it should not be made public immediately.

## What to Include

Please provide the following information when reporting a security issue:

- Clear description of the issue and affected components
- Steps to reproduce
- Expected vs actual behavior
- Affected version(s)
- Proof-of-concept, if available
- Environment details (browser, deployment environment, etc.)

## Supported Releases

Security fixes are provided for the current major release series and any actively maintained release branches. See the [release documentation](https://docs.sjrslms.in/deployment/version-and-release-management) for details.

## Responsible Disclosure

Please do not publicly disclose security vulnerabilities before they have been fixed and released. This protects users and gives maintainers time to address the issue safely.

For detailed reporting guidelines, expected response times, and security best practices, see the [full security policy](https://docs.sjrslms.in/policies/security-policy).
